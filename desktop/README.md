## 🎯 产品定位

**搭言** = 聊天军师 + 话术库 + 知识库 + 自定义词库

```
┌─────────────────────────────────────────────────────────────┐
│                                                              │
│   搭言（dy-as）           搭言·输入                        │
│   ┌─────────────┐         ┌─────────────┐                 │
│   │  话术库管理  │◄──────►│  语音输入   │                 │
│   │  知识库管理  │  调用   │  快捷调用   │                 │
│   │  语料库管理  │         │  文字润色   │                 │
│   │  联系人管理  │         │  智能补全   │                 │
│   │  自定义词库  │         │             │                 │
│   └─────────────┘         └─────────────┘                 │
│                                                              │
│   定位：大脑（管理，重型）   定位：手脚（输入，轻量）        │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

> 💡 **为什么分开？** Typeless 只做输入端，无法管理大量话术和词库。搭言负责「大脑」，搭言·输入负责「手脚」，各司其职。

## ✨ 核心功能

### 🗣️ 搭言·输入 联动
与搭言·输入（语音输入工具）无缝联动：
- **话术库共享**：搭言管理的话术库可被搭言·输入调用
- **知识库接入**：联系人信息、行业话术实时同步
- **自定义词库**：解决 Typeless 产品词识别错误问题
- **文字拦截润色**：敏感词过滤 + 违禁词替换

### 🧠 军师分析
智能分析聊天上下文，提供深度洞察：
- **意图识别**：AI 分析对方深层想法和动机
- **策略建议**：告诉你「应该做」和「不应该做」的事
- **胜率评估**：可视化展示当前关系推进状态
- **推荐回复**：生成多条高质量回复候选
- **综合诊断**：性格标签、风险分析、下一步行动建议

### 📚 话术库
专业聊天话术知识库：
- **分类管理**：工作、情感、商务、日常等多分类
- **快速搜索**：关键词全文检索
- **批量导入**：支持 JSON 格式批量导入话术
- **默认模板**：内置常用优质话术模板

### 🔤 自定义词库（Typeless 痛点终结者）
解决 Typeless 产品名词识别错误问题：
- **产品词库**：品牌名、产品名、型号规格、行业术语
- **联系人词库**：客户名、地名、公司名
- **智能联想**：输入时自动提示正确的产品词
- **语音适配**：让 ASR 正确识别专有名词读音
- **一键导入**：从 Excel/JSON 批量导入词库

> 💡 **痛点场景**：Typeless 经常把「搭言」识别成「打眼」，把「Tauri」识别成「套瑞」。自定义词库让 ASR 学会认识你的产品。

### 💼 销售/客服话术实时转译
**核心场景**：当销售或客服使用输入法时，口语自动转标准话术，避免说错、说乱。

| 场景 | 销售人员口语 | 系统转换 |
|-----|-------------|---------|
| 客户询价 | 「嗯...那个...多少钱来着」 | 「您好，请问这款产品的价格是？」 |
| 报价表达 | 「大概可能得...三千左右吧」 | 「这款产品定价为 2999 元/套」 |
| 催款沟通 | 「那个...你那个钱能不能...」 | 「张总，您好，请问上次合作的款项...」 |
| 售后处理 | 「这个...可能...是我们的问题」 | 「非常抱歉，这是我们的工作失误...」 |

**拦截规则**：
- 口头禅过滤：「嗯」「啊」「那个」「吧」
- 不专业表达替换：「可能」→「确定」
- 违禁词替换：平台禁用词 → 合规表达

### 🎭 用户画像 + 回答标准（人设管理）
统一团队话术风格，保证「人设不崩塌」

| 功能 | 说明 | 示例 |
|-----|------|------|
| **用户画像** | 定义角色身份、性格、专业度 | 客服小美 / 销售总监 / 企业微信客服 |
| **回答标准** | 统一话术风格、语气、敬语 | 正式/亲切/专业/幽默 |
| **人设设定** | 设定角色背景、立场、专业领域 | 「3年经验的SaaS产品专家」 |
| **违规检测** | 自动检测不符合人设的回复 | 语气不当/用词错误/超出权限 |

**人设设定示例**：

```
┌────────────────────────────────────────────────────────┐
│  【用户画像设定】                                        │
├────────────────────────────────────────────────────────┤
│                                                        │
│  角色名称：搭言客服-小雨                               │
│  角色定位：专业、热情、耐心                            │
│  语气风格：亲切但不随便，正式但不冷漠                   │
│                                                        │
│  【回答标准】                                          │
│  ├─ 开头：「您好，我是搭言客服小雨，很高兴为您服务~」  │
│  ├─ 结尾：「祝您生活愉快，有问题随时联系~」           │
│  ├─ 禁用语：「不知道」「你问别人」「不行」             │
│  └─ 必须用：「您好」「请问」「帮您」「稍等」          │
│                                                        │
│  【专业背景】                                          │
│  ├─ 熟悉产品：搭言、搭言·输入全系列                   │
│  ├─ 价格政策：标准报价体系，可申请折扣                │
│  └─ 售后政策：7天无理由、终身维护                     │
│                                                        │
└────────────────────────────────────────────────────────┘
```

**人设应用效果**：

| 员工原始回复 | 人设检测 | 标准回复 |
|-------------|---------|---------|
| 「不知道」 | ❌ 违规 | 「您好，这个我需要帮您确认一下，稍等」 |
| 「不行」 | ❌ 违规 | 「非常抱歉，目前暂时无法支持，您的需求已记录」 |
| 「随便你」 | ❌ 违规 | 「好的，如果您有其他问题，随时联系我」 |
| 「嗯」 | ❌ 违规 | 「好的，请问还有什么可以帮您？」 |

**团队管理**：
- 管理员可预设多套人设模板
- 不同产品线/部门使用不同人设
- 新员工直接选用人设，快速上手
- 人设执行统计，定期优化话术

## 🛠️ 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| **桌面容器** | Electron 28 | 跨平台桌面应用支持 |
| **前端框架** | Vue 3 Composition API | 现代化响应式开发 |
| **构建工具** | Vite 5 | 极速开发体验 |
| **路由管理** | Vue Router 4 | SPA 单页应用路由 |
| **本地存储** | Dexie.js | IndexedDB 封装，纯 JS 无原生依赖 |
| **AI 引擎** | OpenAI API | 兼容所有 OpenAI 格式接口 |
| **语言** | TypeScript | 类型安全 |
| **工具库** | Lodash ES / Day.js | 工具函数与日期处理 |
| **OCR** | Tesseract.js | 纯 JS 文字识别（待启用） |

## ⚙️ 配置说明

### AI 配置
在应用设置页面配置：
- **API Key**：你的 OpenAI 兼容接口密钥
- **Base URL**：API 端点地址（支持第三方兼容接口）
- **Model**：使用的模型名称（如 gpt-3.5-turbo）

### 全局快捷键
- `Ctrl + Shift + D`：触发屏幕识别（OCR 功能待启用）
