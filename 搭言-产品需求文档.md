## 🎯 01. 产品决策背景

### 1.1 为什么做这个产品（Why Now）

**市场机会：**
- 微信月活13亿+，几乎所有职场人每天都在用微信沟通
- AI大模型成熟，生成式回复质量达到可用水平
- 市场上同类产品（如微信输入法智能帮写）功能太弱，没有深度
- 目前没有专门针对聊天场景的、策略级的AI助手
- 用户对"封号"极度敏感，纯辅助定位产品有巨大优势

**痛点验证：**
1. ❓「不知道怎么回」：65%用户承认曾对着微信输入框发呆超过5分钟
2. ⏰「打字太慢」：销售/客服每天打字超过3小时，重复话术占比80%
3. 😰「怕说错话」：83%用户后悔过微信上说错话，尤其是跟老板/客户沟通
4. 🔒「怕封号」：第三方微信工具封号率极高，用户不敢用

**竞争格局：**
- 微信输入法「智能帮写」：功能太基础，只有简单补全
- 各类自动回复机器人：风险高，容易封号，策略简单
- ChatGPT网页版：需要复制粘贴，流程割裂，没有场景化
- ✅ **市场空白**：安全、不封号、有策略深度的AI聊天助手

---

### 1.2 产品愿景与目标

**产品愿景：**
> 成为1亿人的智能聊天副驾驶，让每一次沟通都恰到好处

**v1.0 核心目标：**
| 指标 | 目标值 | 计算方式 |
|------|--------|----------|
| 日活用户 | 10,000 | 后台统计（仅统计激活数，不收集用户数据） |

| 次日留存 | >40% | 第2天还打开软件的用户占比 |
| 人均日使用次数 | >15次 | 平均每个用户每天点击生成回复次数 |

**商业目标：**
- v1.0上线后3个月内达到月营收10万
- 12个月内达到月营收100万
- 成为国内微信聊天助手品类第一名

## 👤 02. 用户画像分析

### Persona 1：销售小李（核心付费用户）
```
基本信息：男，28岁，To B销售，月薪15k

典型场景：
- 每天跟30+客户聊微信
- 客户砍价、压周期、提需求，每一条回复都要琢磨半天
- 最怕跟老板汇报工作，措辞要想半小时

核心诉求：
1. 快速生成专业得体的商务回复
2. 客户砍价怎么回？给我几套话术方案
3. 常用话术一键调用，不用反复打字

付费意愿：★★★★★  ¥59/月 = 2杯奶茶钱 = 多签一个单赚回来
```

### Persona 2：客服小张（高频用户）
```
基本信息：女，24岁，电商客服，月薪6k

典型场景：
- 同时登3个微信，每天接待上百个客户
- 90%问题是重复的：快递什么时候发？怎么退货？有优惠吗？
- 下班手都酸了，打字打到腱鞘炎

核心诉求：
1. 常用话术一键发送，自定义分类
2. 客户问什么，自动匹配最佳回复
3. 多模态识别，客户发截图也能看懂

付费意愿：★★★★☆  公司可以报销
```

### Persona 3：社恐小王（情感用户）
```
基本信息：男，25岁，程序员，母胎solo

典型场景：
- 跟喜欢的女生聊天，一句话想半小时
- 得罪人了不知道怎么道歉
- 同事提不合理请求，不知道怎么拒绝

核心诉求：
1. 给我几个选项，选一个最像我会说的
2. 告诉我对方这句话到底是什么意思
3. 教教我怎么聊能推进关系

付费意愿：★★★☆☆  有用就愿意花钱
```

### Persona 4：老板老王（安全需求用户）
```
基本信息：男，45岁，私企老板

典型场景：
- 经常收到"我是你朋友介绍个项目"的诈骗信息
- 手下员工被骗过，公司损失几十万
- 微信里人太多，分不清谁是好人谁是骗子

核心诉求：
1. 识别诈骗风险，高风险弹窗提醒
2. 分析对方靠不靠谱，什么来头
3. 绝对安全，不能泄露我的聊天记录

付费意愿：★★★★★  安全无价
```

## 📱 04. 详细功能需求

### 4.1 主界面交互规范

#### 4.1.1 窗口吸附行为
```
触发时机：
1. 软件启动时，自动查找微信窗口
2. 微信窗口位置/大小变化时，自动跟随调整

行为细节：
- 吸附到微信窗口右侧，间距 8px
- 高度与微信窗口保持一致
- 宽度固定 380px（可用户手动调整）
- 微信最小化时，搭言也自动最小化
- 微信关闭时，搭言提示"未检测到微信窗口"

异常处理：
- 没找到微信：显示友好提示 + 手动吸附按钮
- 多个微信：让用户选择吸附到哪个
```

### 4.3 AI引擎详细需求

#### 4.3.1 回复质量标准
| 指标 | 要求 |
|------|------|
| 生成速度 | 90%情况下 < 3秒，最长不超过8秒 |
| 可用性 | 生成的3个选项中，至少有1个可以直接发送 |
| 相关性 | 回复与上下文强相关，不答非所问 |
| 自然度 | 读起来像真人说的，没有明显AI腔 |
| 安全性 | 绝对不生成违法、违规、政治敏感内容 |

#### 4.3.2 提示词架构设计
```
系统级Prompt（固定）：
  角色设定：你是一个专业的微信聊天助手
  风格要求：像真人一样自然，不要AI腔
  安全要求：绝对不生成违法违规内容
  输出格式：每次生成3个不同风格的选项

场景级Prompt（按功能切换）：
  正常回复模式 / 润色模式 / 军师分析模式 / 反诈检测模式

用户级Prompt（个性化）：
  用户说话风格特征（学习后注入）
  用户常用语、口头禅
  用户的职业、身份标签
```

## 🔐 06. 非功能需求

### 6.1 性能要求
| 指标 | 目标值 | 备注 |
|------|--------|------|
| 冷启动时间 | < 2秒 | 从双击图标到界面可用 |
| AI回复生成 | < 3秒 | 90%情况下 |
| 内存占用（空闲） | < 150MB | 后台挂着不卡电脑 |
| 内存占用（工作中） | < 300MB | 生成回复时峰值 |
| CPU占用（空闲） | < 1% | 几乎不感知 |
| CPU占用（工作中） | < 5% | 不影响其他软件 |
| 安装包大小 | < 80MB | 下载快 |
| OCR识别速度 | < 1秒/张 | 本地离线OCR |

### 6.2 安全要求（最高优先级）
✅ **零数据上传原则**
- 聊天内容、用户输入、联系人信息 100% 本地处理
- 只在调用大模型API时发送必要的生成内容
- 绝对不向我们自己的服务器上传任何用户聊天数据

✅ **防封号红线**
- ❌ 绝对不注入微信进程
- ❌ 绝对不调用微信私有API
- ❌ 绝对不模拟点击发送按钮（发送必须用户手动按回车）
- ✅ 只使用公开的Windows系统API

✅ **数据安全**
- 本地SQLite数据库 AES-256 加密
- 用户配置文件加密存储
- 激活信息绑定硬件，不可复制
- 卸载时一键擦除所有本地数据

### 6.3 兼容性要求
- **Windows**: Windows 10 1809+ / Windows 11（覆盖95%以上用户）
- **微信PC版**: 支持 3.7.0 ~ 最新版，向前兼容3个大版本
- **高分屏**: 支持 100%/125%/150%/200% DPI 缩放，界面不模糊不错位

### 6.4 稳定性要求
- 连续运行72小时不崩溃
- 异常退出后自动重启恢复
- 每天崩溃率 < 0.1%
- AI调用失败率 < 1%

## 📊 07. 数据埋点与指标体系

### ⚠️ 重要前提：隐私第一
**我们不收集任何用户聊天内容、不跟踪用户行为、不上传任何个人信息。**

只统计以下匿名数据，上报时只带随机设备ID，不带任何个人标识：
```
1. 激活相关：激活成功/失败、激活码类型（PRO/SVIP）
2. 性能相关：启动耗时、AI生成平均耗时、崩溃率
3. 功能使用：每天生成回复次数、各功能点击量（匿名计数）
4. 版本分布：各版本号的用户分布
```

### 核心北极星指标
> **人均日生成回复次数**

### 看板指标体系
| 指标类型 | 具体指标 |
|----------|----------|
| **增长指标** | 日激活数、日活、7日留存、30日留存 |
| **商业指标** | 付费转化率、ARPU、月营收、续费率 |
| **功能指标** | 各功能使用次数分布、军师模式使用率 |
| **性能指标** | 崩溃率、平均生成耗时、成功率 |
