## 🛠️ 技术选型

### 【桌面端】Windows + macOS
| 技术栈 | 选型 | 理由 |
|--------|------|------|
| 前端框架 | **Electron + Vue 3 + Vite** | 成熟稳定，跨平台，生态完善，开发效率高 |
| UI组件库 | **Naive UI** / **Element Plus** | 现代化，主题定制方便 |
| 系统能力 | **Node.js 原生模块** | 调用Windows API、剪贴板、通知监听 |
| 本地数据库 | **SQLite + better-sqlite3** | 高性能，嵌入式，不需要单独安装 |
| 本地OCR | **PaddleOCR** / **Tesseract** | 离线可用，识别率高，支持中文 |
| 打包工具 | **electron-builder** | 支持自动更新、代码签名、安装包制作 |

### 【移动端】iOS + Android
| 技术栈 | 选型 | 理由 |
|--------|------|------|
| 跨平台框架 | **Flutter 3.x** | 性能接近原生，UI一致性好，热重载开发快 |
| 状态管理 | **Riverpod / Bloc** | 可预测，可测试，适合中大型项目 |
| 本地数据库 | **sqflite** (SQLite) | 与桌面端数据格式可兼容 |
| 系统集成 | **原生通道 (Method Channel)** | 调用剪贴板、分享、通知等系统能力 |

### 【后端服务 (轻量级)】
| 技术栈 | 选型 | 理由 |
|--------|------|------|
| 服务端 | **Node.js + NestJS** | TypeScript 全栈，前后端语言统一 |
| 授权服务 | **自定义激活码系统** | 机器码绑定，离线激活<br>📝 备注：未来计划采用在线授权系统，支持实时验证、账号绑定、订阅管理等功能 |
| 更新服务 | **简单静态CDN** | 版本检查、增量更新 |
| 数据统计 | **无数据收集原则** | 只统计激活数、版本号，不收集用户聊天内容 |

### 4. 授权与激活系统
```typescript
class LicenseManager {
  // 获取机器码
  getMachineId(): string

  // 验证激活码
  verifyLicense(key: string): LicenseInfo

  // 在线激活
  async activateOnline(key: string): ActivateResult

  // 离线激活
  activateOffline(licenseFile: File): boolean

  // 检查权限
  hasPermission(feature: Feature): boolean
}
```

**安全机制**：
- 机器码 = CPU序列号 + 硬盘序列号 + MAC地址（三选二哈希）
- 激活码采用RSA非对称加密，本地验签，不需要连网
- 支持设备转移，每月限1次

## 🔒 安全架构设计（核心竞争力）

### 1. 零数据上传原则
- ❌ 聊天内容不上传服务器
- ❌ 用户输入不发送到我们的后端
- ❌ 不记录联系人信息
- ✅ 所有AI调用直接走大模型官方API（或用户自己的Key）

### 2. 本地数据加密
- SQLite数据库 **AES-256加密**
- 用户配置文件加密存储
- 激活信息绑定机器硬件
- 卸载时可一键擦除所有数据

### 3. 防封号保障
- ✅ 不注入微信进程
- ✅ 不调用微信私有API
- ✅ 只使用公开的Windows系统API
- ✅ 模拟人工操作，有随机延时
- ✅ 用户最终按回车发送，完全是人工行为

## 📲 移动端核心能力实现

### iOS方案（受限较多，推荐做辅助式）
- **扩展方式**：做键盘扩展，系统任何地方打字都能用
- **共享方式**：复制内容到剪贴板，打开搭言自动生成回复
- **快捷指令**：支持iOS快捷指令，一键生成

- **通知监听**：收到微信通知自动提取内容
- **悬浮窗**：聊天界面弹出悬浮按钮，一键生成回复
- **无障碍服务**：可选，自动粘贴功能（需用户授权）

## 📌 技术决策原则

1. **用户隐私优先**：能本地做的绝不走云端，能不上传的绝对不上传
2. **性能优先**：宁可多写原生代码，也不要为了跨平台牺牲体验
3. **简单为王**：后端越简单越好，最好大部分功能纯本地就能用
4. **灰度迭代**：复杂功能先做MVP，用起来再优化，不要一开始就做完美架构
5. **安全第一**：所有可能导致封号的风险点，提前评审，宁可砍掉也不冒险
