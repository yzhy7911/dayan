# 军师多轮对话功能设计

## 📋 当前问题

### 问题描述

❌ **当前实现**：
- 每次分析都是独立的单次请求
- 没有对话历史记录
- 没有目标管理
- 无法结合上下文理解对话进展

❌ **用户体验**：
```
用户：客户砍价
AI：分析意图+策略

用户：但他还要优惠
AI：从头分析，不记得上文

问题：
1. 不记得客户的反应
2. 不记得之前的策略
3. 无法跟踪对话进展
4. 每次都是新的开始
```

---

## 🎯 设计目标

### 核心需求

✅ **多轮对话支持**：
- 记住对话历史
- 支持追问和补充
- 结合上下文分析

✅ **多目标管理**：
- 为每个场景/客户创建独立目标
- 目标间相互隔离
- 方便管理多个对话

✅ **上下文理解**：
- AI 能记住之前的分析和建议
- 跟踪对话进展
- 理解对话发展脉络

---

## 💡 解决方案

### 方案一：简化版（快速实现）

#### 数据结构

```typescript
interface CoachGoal {
  id: string
  name: string
  goal: string
  status: 'active' | 'completed' | 'archived'
  messages: CoachMessage[]
  createdAt: number
  updatedAt: number
}

interface CoachMessage {
  id: string
  role: 'user' | 'assistant'
  content: string
  analysis?: {
    intent: string
    emotion: string
    winRate: number
    timestamp: number
  }
  timestamp: number
}
```

#### 功能

1. **目标列表**
   - 创建新目标（客户名称/场景描述）
   - 切换目标
   - 查看历史目标

2. **对话输入**
   - 每次输入追加到当前目标的messages
   - 支持追问："但是他又说xxx"

3. **上下文分析**
   - 发送完整历史给AI
   - AI结合上下文分析
   - 更新分析结果

## 💡 技术实现

### 数据流

```
用户输入 → 追加到当前目标 → 保存到IndexedDB 
      ↓
发送完整历史给AI → AI分析 → 更新分析结果 
      ↓
显示胜率/策略/话术 → 用户选择话术 → 复制/发送
```

### AI Prompt 增强

```typescript
// 增强的上下文理解
const COACH_PROMPTS_WITH_CONTEXT = {
  system: `你是聊天军师，记住完整对话历史，给出连贯的分析和建议。
  
  【上下文理解】
  1. 记住之前的对话内容
  2. 理解对话进展和转折
  3. 基于历史给出针对性建议
  
  【对话历史】
  {context}
  
  【当前输入】
  {currentInput}
  
  【分析要求】
  1. 结合历史分析当前情境
  2. 评估对话进展（顺利/僵持/恶化）
  3. 给出针对性建议`,
}
```
