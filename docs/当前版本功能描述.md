# 搭言当前版本功能描述

> 文档日期：2026-04-30  
> 适用范围：当前桌面端版本，基于现有 Electron + Vue 实现梳理  
> 产品定位：面向微信等即时沟通场景的桌面沟通工作台，帮助用户快速生成回复、分析对话局势、整理话术资产，并在本地完成截图识别与语音听写辅助。

## 1. 产品概述

搭言是一款桌面端 AI 沟通助手。当前版本围绕“回复、军师、联系人、润色、话术、历史、设置”七个主入口展开，目标是在用户面对难回复、难判断、需要专业表达或需要复用常用话术时，提供一个贴近微信工作流的辅助面板。

当前版本已经具备较完整的单机桌面工作台形态：

- 支持 AI 智能回复，多种风格和回复目标可选。
- 支持截图 OCR，能够从粘贴图片中提取聊天记录。
- 支持聊天军师，对目标会话进行意图、风险、策略和推荐话术分析。
- 支持本地离线语音听写，用于通话中转述对方说法并获取 AI 建议。
- 支持话术本管理，可新增、搜索、分类、导入和导出话术。
- 支持联系人画像、聊天历史沉淀和基础分析。
- 支持文本润色、历史记录、设置、快捷键、窗口吸附等桌面能力。

## 2. 整体信息架构

当前应用左侧为固定导航栏，顶部为当前页面标题和窗口控制区，主内容区承载不同功能页面。

主导航包括：

| 入口 | 当前名称 | 主要用途 |
| --- | --- | --- |
| 回复 | 智能回复 | 粘贴聊天、截图或文字，生成可发送回复 |
| 军师 | 聊天军师 | 分析对话局势、判断意图、给出策略和回复建议 |
| 联系人 | 联系人画像 | 管理联系人资料、标签、聊天记录和画像分析 |
| 润色 | 文本润色 | 把用户已有文字改写为更合适的表达 |
| 话术 | 话术本 | 管理可直接复用的标准话术 |
| 历史 | 历史记录 | 查看、搜索、收藏、删除生成过的回复 |
| 设置 | 系统设置 | 管理模型配置、窗口行为、快捷键、数据和授权 |

左侧品牌区已使用现有应用图标，并展示“搭言 / 工作台”。顶部标题栏显示当前功能页名称，避免重复堆叠品牌名。

## 3. 智能回复

智能回复是当前版本的核心入口，用于把聊天上下文、截图 OCR 内容或用户直接输入的问题，整理成可发送的候选回复。

### 3.1 输入能力

支持三类输入：

- 直接输入文本：用户可以手动输入对方消息、当前局面或补充说明。
- 粘贴聊天内容：可将微信聊天文本粘贴到输入框。
- 粘贴截图：通过 OCR 提取截图中的当前会话内容。

输入区当前会区分两种使用意图：

- 图片/聊天记录识别：作为主要上下文输入。
- 用户直接输入文本：可作为当前局面描述、补充说明或要回复的内容。

### 3.2 截图 OCR 与聊天清洗

当前版本已接入本地 PaddleOCR 相关能力，支持识别粘贴图片中的文字，并对聊天截图做清洗。

主要清洗目标包括：

- 尽量只提取当前会话内的聊天内容。
- 区分“对方”和“我”的发言。
- 过滤应用左侧菜单、标题栏、按钮、输入框占位、系统说明等干扰文字。
- 过滤头像中的文字。
- 过滤语音气泡时长，例如 `16"`、`47"` 这类非聊天正文。
- 过滤图片、视频等非文本消息占位信息。

### 3.3 回复风格

当前支持多种回复风格：

- 全部风格
- 友好
- 正式
- 幽默
- 简洁
- 共情

用户选择“全部风格”时，系统会生成多条不同风格的候选回复。选择单一风格时，重点输出该风格下的回复。

### 3.4 回复目标

当前支持按回复目标生成更有方向性的内容：

- 自然接话
- 安抚
- 解释
- 拒绝
- 推进
- 收尾

该能力用于解决“同样一句话，用户想达到不同沟通目的”的问题。例如客户质疑时可以选择解释，关系紧张时可以选择安抚，谈判推进时可以选择推进。

### 3.5 生成结果

回复结果以候选回复形式展示。每条回复可用于后续复制、发送或沉淀到历史记录。

当前回复生成已经做过自然感优化，减少机械句式，例如避免过多出现“建议你可以回复”“根据你说的”等解释型开头，更接近微信里可以直接发送的一句话。

### 3.6 当前边界

当前回复页还没有正式接入“知识库依据检索”和“话术本优先匹配”。也就是说，它已经能生成自然回复，但还没有完成“客户专业问题先查资料、再生成答复”的闭环。这是后续版本的重要升级点。

## 4. 聊天军师

聊天军师用于处理比“回一句”更复杂的沟通局面。它不仅生成回复，还会分析对方意图、当前风险、推荐策略和下一步表达。

### 4.1 目标管理

军师页支持创建聊天目标。目标可以理解为一次具体的沟通任务，例如：

- 推进客户成交
- 挽回一段关系
- 解释一个误会
- 判断对方真实态度
- 处理客户异议

每个目标下可以沉淀多轮消息和分析结果。

### 4.2 聊天记录输入

军师页支持用户输入或粘贴聊天记录，并围绕当前目标进行分析。

当前还支持从剪贴板导入内容，便于用户复制微信聊天后直接整理到军师页面。

### 4.3 技能选择

当前内置军师技能包括：

- 恋爱军师
- 金牌销售

技能会影响 AI 分析视角和推荐话术风格。例如恋爱军师更关注情绪、关系状态和推进节奏；金牌销售更关注客户意图、异议处理、风险承诺和成交推进。

### 4.4 分析维度

军师分析结果包含多维信息：

- 胜率或当前局势判断
- 表层意图
- 真实意图
- 情绪状态
- 真实需求
- 风险点
- 应该做
- 不要做
- 推荐语气
- 推荐回复

结果区域会把“策略建议”和“推荐话术”分开展示，让用户既能理解局势，也能快速拿到可用表达。

### 4.5 历史面板

军师页已有历史记录面板，用于查看既往目标和分析内容。当前面板支持点击展开和收缩，方便在较小窗口中控制显示空间。

### 4.6 通话军师

当前版本已加入通话按钮，支持用户在选择军师技能后，以语音方式转述对方说了什么。

通话军师当前能力：

- 支持打开通话面板。
- 支持选择当前说话人：对方 / 我。
- 支持本地离线语音识别状态展示。
- 支持连续听写，将语音转成文字。
- 支持把转写内容加入当前军师上下文。
- 支持自动分析开关，用于在听写过程中触发建议。

语音识别采用本地离线方案，依赖用户本机模型文件，不依赖云端语音服务。这样可以降低成本并保护隐私。

### 4.7 当前边界

通话军师目前以“语音转文字 + 文本建议”为主，不强制朗读 AI 回复。识别率取决于本地模型质量、麦克风输入、环境噪声和说话清晰度。

军师页当前还未正式接入知识库检索。后续应让军师在分析销售或恋爱场景时，引用相关业务资料、禁忌事项、关系笔记或标准口径。

## 5. 联系人画像

联系人页面用于管理聊天对象信息，并沉淀与某个对象相关的背景、标签和聊天记录。

### 5.1 联系人管理

当前支持：

- 创建联系人
- 编辑联系人基础信息
- 删除联系人
- 维护备注
- 维护标签

### 5.2 聊天记录沉淀

联系人可以关联聊天记录。用户可以把重要对话保存到某个联系人下，便于后续分析或回看。

### 5.3 画像分析

联系人页具备基础分析展示能力，包括：

- 性格倾向
- 沟通风格
- 情绪状态
- 沟通建议
- 最佳回复时间等辅助信息

### 5.4 当前边界

联系人画像还没有深度打通回复页和军师页。后续可让回复和军师自动读取联系人偏好、禁忌、历史矛盾、客户阶段等信息。

## 6. 文本润色

文本润色用于优化用户已经写好的内容。它不是从零生成回复，而是把原文改得更得体、更清楚或更符合场景。

### 6.1 输入与输出

用户输入原文后，可选择润色方向，系统输出改写后的文本。

### 6.2 适用场景

典型场景包括：

- 把语气改得更友好
- 把表达改得更正式
- 把长句变得更简洁
- 把拒绝说得更委婉
- 把销售表达改得更专业

### 6.3 当前边界

润色页当前主要基于用户输入文本工作，还没有和联系人画像、知识库或话术本深度联动。

## 7. 话术本

当前“话术”页面本质上是话术本，用于管理可直接复用的标准回答。

它和后续规划中的“知识库”应有所区分：

- 话术本：标准回答，可以直接发。
- 知识库：事实资料、背景信息、规则依据，不一定能直接发。

### 7.1 话术管理

当前支持：

- 新建话术
- 删除话术
- 复制话术
- 按分类浏览
- 按关键词或内容搜索

每条话术包含：

- 分类
- 关键词
- 话术内容
- 创建时间

### 7.2 分类

当前内置分类包括：

- 工作
- 情感
- 商务
- 日常

### 7.3 批量导入

当前支持通过文件导入话术资产。

支持格式：

- JSON
- CSV

文件选择器当前也允许选择 TXT，但实际解析逻辑主要面向 JSON 和 CSV；TXT 若不是合法 JSON，不会被识别为有效话术数据。

导入流程包括：

- 选择文件
- 解析数据
- 预览可导入话术
- 处理重复关键词
- 确认导入

### 7.4 导出

当前支持将话术本导出为 JSON 文件，便于备份或迁移。

### 7.5 当前边界

当前话术本主要依赖用户主动新增、导入和导出。还没有实现从知识库资料中自动提取标准问题和标准回答，也没有在回复页中优先命中话术本。

## 8. 历史记录

历史记录用于沉淀用户生成过的回复，方便回看和复用。

### 8.1 保存内容

历史记录会保存：

- 输入文本
- 输入图片信息
- 所选风格
- 生成的回复列表
- 是否收藏
- 创建时间
- 更新时间

### 8.2 使用能力

当前支持：

- 查看历史回复
- 搜索输入或回复内容
- 收藏重要回复
- 删除单条历史
- 清空历史

### 8.3 自动清理

系统设置了历史记录数量保护，避免数据无限增长。当前策略会保留收藏内容，并清理较旧的非收藏记录。

## 9. 设置与桌面行为

设置页用于管理应用运行环境、模型配置、快捷键和数据能力。

### 9.1 AI 模型配置

当前支持 OpenAI 兼容接口配置：

- API 服务商
- API Key
- API 地址
- 模型名称
- 测试连接
- 保存配置
- 恢复默认

这意味着用户可以接入 OpenAI 兼容的第三方模型服务。

### 9.2 自定义系统提示

设置页支持自定义回复 Prompt。用户可以输入希望模型长期遵循的表达要求；留空时使用系统内置默认 Prompt。

### 9.3 快捷键

当前支持配置快捷键：

- 显示 / 隐藏窗口
- 快速回复
- 快速润色

快捷键可在设置页修改、重置和保存。

### 9.4 窗口行为

当前支持：

- 窗口吸附到微信窗口
- 窗口置顶
- 自定义最小化与关闭按钮
- 无边框窗口

在 macOS 上，吸附能力依赖辅助功能权限和 AppleScript 检测。

### 9.5 授权信息

设置页包含授权状态展示和激活入口。当前界面中会展示未激活或已激活状态。

### 9.6 数据同步

当前支持：

- 导出备份
- 导入恢复

用于迁移或备份本地数据。

## 10. 本地识别能力

当前版本包含两类本地识别能力。

### 10.1 本地 OCR

OCR 用于识别聊天截图、图片中的文字。当前打包配置已包含 OCR 模型文件，并在应用启动后延迟注册 OCR 服务，避免 native 依赖阻塞窗口创建。

使用的相关能力包括：

- PaddleOCR
- onnxruntime-node
- sharp
- 本地 OCR 模型文件

### 10.2 本地 ASR

ASR 用于语音听写。当前版本采用 Vosk 本地离线识别方案。

特点：

- 不依赖云端语音接口。
- 模型文件放在本地 `models/asr` 目录。
- 打包时会把 ASR 模型和 native 运行库一并带入应用资源目录。
- 识别结果用于通话军师的文字上下文。

## 11. 数据存储

当前版本主要使用本地存储。

| 数据类型 | 存储方式 |
| --- | --- |
| AI 配置 | localStorage |
| 回复历史 | Dexie.js / IndexedDB |
| 话术本 | Dexie.js / IndexedDB |
| 联系人 | Dexie.js / IndexedDB |
| 军师目标与消息 | Dexie.js / IndexedDB |
| 备份文件 | 用户主动导出 |

当前 README 中部分旧描述仍提到 SQLite 和 AES 加密，但当前代码实际以 Dexie.js / IndexedDB 为主。

## 12. 隐私与安全

当前版本的隐私特点：

- 截图 OCR 在本地进行。
- 语音识别使用本地离线模型。
- 历史、话术、联系人、军师目标主要保存在本机 IndexedDB。
- AI 回复和军师分析会调用用户配置的大模型接口，因此发送给模型的数据取决于用户实际触发的生成或分析行为。

需要明确的是：

- 本地识别不等于所有功能完全离线。
- AI 生成仍需要外部模型服务。
- 用户应避免把敏感信息发送给不可信的模型服务商。

## 13. 打包与运行

当前桌面端基于 Electron 28、Vue 3、Vite 构建。

已配置平台：

- macOS DMG
- Windows NSIS

当前已验证：

- `vite build` 可通过。
- `electron-builder --dir` 可生成 macOS 应用目录。
- 打包后的 macOS 应用可正常启动窗口。
- OCR、ASR 相关模型和 native 依赖已配置为打包资源或 asar unpack。

当前发布前仍需处理：

- macOS 正式签名。
- DMG 完整安装包验证。
- Windows 打包验证。
- 麦克风权限引导和真实通话场景回归测试。

## 14. 当前版本主要短板

当前版本功能已形成完整工作台，但仍存在几个明确短板：

### 14.1 知识库尚未独立成型

当前“话术”页面承担了部分资料管理职责，但严格来说它更像“话术本”。真正的知识库还没有拆出来。

后续应区分：

- 话术本：标准回答库，直接发。
- 知识库：资料依据库，供 AI 生成和军师分析时参考。

### 14.2 回复页未接入话术本和知识库

原型中回复页包含“知识库匹配”和“知识库命中回复卡”。当前实现还没有完成这条链路。

后续应实现：

- 先匹配话术本。
- 再匹配知识库。
- 最后再做纯 AI 临场生成。

### 14.3 军师页未接入知识库

军师当前主要基于聊天上下文和技能 Prompt 分析，还没有引用销售资料、恋爱笔记、客户背景等知识内容。

### 14.4 话术本仍依赖手动维护

当前话术本依赖新增、导入和导出。后续可以从知识库中提炼标准问题和标准回答，由用户审核后加入话术本。

### 14.5 本地 ASR 识别率依赖模型

当前 Vosk 小模型可用，但识别率有限。后续可以提供更好的模型选择、模型下载引导或用户自选模型能力。

## 15. 当前版本一句话总结

当前版本的搭言已经是一款可用的桌面 AI 沟通工作台：它能识别聊天截图、生成自然回复、分析复杂对话、进行本地语音听写、管理话术和历史，并具备基础联系人画像与桌面集成能力。

下一阶段的核心升级，不应继续堆独立页面，而应围绕“话术本 + 知识库 + 回复 + 军师”的联动，把搭言从“会生成回复”升级为“能基于资料和场景给出可靠答复”。
