### 问题 2：better-sqlite3 版本不匹配
**原因**：Node.js 原生模块编译版本与 Electron 不兼容
- pnpm install 按系统 Node.js 版本编译（NODE_MODULE_VERSION 115+）
- Electron 28 内置 Node.js 18（NODE_MODULE_VERSION 109）
- 版本不匹配导致启动报错

**已修复**：
- ✅ 添加 `better-sqlite3` 回依赖
- ✅ 添加 `electron-rebuild` 开发工具
- ✅ package.json 添加 `pnpm run rebuild` 脚本
- ✅ package.json 添加 `postinstall` 钩子（自动执行）
- ✅ 数据库代码添加降级容错：SQLite 失败自动切内存模式
- ✅ 添加 `isMemoryMode()` 状态检测

**本地修复步骤**：
```bash
cd desktop

# 1. 清理旧依赖
rm -rf node_modules pnpm-lock.yaml

# 2. 重新安装（会自动执行 postinstall）
pnpm install

# 3. 如果还有问题，手动重编译
pnpm run rebuild
```

**运行状态检测**：
- 控制台看到 `✅ SQLite 初始化成功` = 正常
- 控制台看到 `⚠️ SQLite 初始化失败，降级到内存模式` = 需要执行 rebuild

### 开发环境说明

- **npm install 网络问题**：当前环境访问 npm 镜像受限，需在本地 Windows 环境安装依赖
- **数据库初始化**：首次启动会自动创建 SQLite 数据库和表结构
- **降级模式**：SQLite 初始化失败自动切到内存模式，不影响开发

### 本地开发步骤（在 Windows 机器上）：

```bash
# 1. 进入项目目录
cd desktop

# 2. 配置镜像源（如果需要）
npm config set registry https://registry.npmmirror.com
set ELECTRON_MIRROR=https://npmmirror.com/mirrors/electron/

# 3. 安装依赖
npm install

# 4. 启动开发模式
npm run dev
```
